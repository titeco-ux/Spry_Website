// THE METHODOLOGY

Mainline. The line every project ships to.

Mainline is what we call our methodology. Borrowed on purpose from trunk-based development — the discipline the best engineering teams already aspire to. We extended it across every station of the factory.

One main branch for how Spry works. No long-lived feature branches. No handoffs between disciplines. No "we'll integrate later." Every phase commits its work back to the same line — and the next project starts from where the last one left off. That's the compounding part.
